## Commit rules

Record the changes made by making changes in accordance with [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)

```
<type>: <description>
```

### Allowed `<type>`

- `docs` - documentation
- `feat` - new feature
- `fix` - bug fix
- `refactor` -  reorganization without breaking changes and new features
- `community` - editing community files such as `README.md`
- `deps` - updating dependencies

### Style for description

- Only English language
- Don't capitalize first letter
- Use imperative, present tense
- Don't use (`.`) in the end
